### PatchMatch

It wasn't at all what I expected! I thought the original PatchMatch was also intended for MVS, but turned out it was for image editing - specifically tools like [Content-aware Fill](https://nigelwaters.photography/blog/content-aware-fill-photoshop/)!
<PostImage src={contentAwareFillImg} alt="content-aware fill" caption="Example of Content-Aware fill. Works funky, but great in some cases!" />

PatchMatch proposed a very efficient way to find **matches** between **patches** (haha) which wasn't bruteforce. Though it was pretty simple - it assumed both patches were ailgned and we could match them pixel-by-pixel with just a single offset value

The algorithm is interesting - I feel like we can draw a parallel to [RANSAC](https://en.wikipedia.org/wiki/Random_sample_consensus) - we're replacing bruteforce searching with some random sampling + fit metric, in PatchMatch we also have very efficient propagation
<PostImage src={patchMatchProcess} alt="patch match process" caption="Example of applying patch match to two images. It tries to reconstruct first image simply by offsetting pixels from the second one" />

But the efficient algorithm is the main thing that PatchMatch did, and it wasn't actually the paper that proposed using these matches for image editing - it was [Summarizing visual data using bidirectional similarity](https://www.wisdom.weizmann.ac.il/~vision/VisualSummary/bidirectional_similarity_CVPR2008.pdf)

### Bidirectional similarity
Core idea - it defined a "image similarity" function which evaluated whether the contents of two images are identical - by compared patches at varying scales. 
<PostImage src={bidirectionalFormula} alt="bidirectional similarity formula" caption="Formula. It's actually *dissimilarity* - i.e. it's larger when two images are more different" />

I.e. if an image has all the same objects but just in a different arrangement, the similarity is satisfied
<PostImage src={bidirectionalExamples} alt="bidirectional similarity examples" caption="Examples showing what completeness and coherence mean here" height="400" />

Then we can do stuff like:
- Image summarization - generate an image that is 3x smaller but contains all the same content
<PostImage src={imageSummarization} alt="image summarization" caption="Showimg image summarization - we downscale by 0.95, then optimize for image similarity" height="400" />
- Photo merging - we just concatenate images and then optimize for similarity to all input images - then the discontinuities become some kind of merging
<PostImage src={imageMerging} alt="image merging" caption="Showimg image merging - we concatenate images, then optimize for similarity to all input images. Discontinuous seams are replaced with something ok" height="400" />
- Image synthesis - we stretch the image & fill in the remaining things optimizing for coherence
<PostImage src={imageSynthesis} alt="image synthesis" caption="Showimg image synthesis" height="400" />

I was reeeeally surprising that such simple formulas lead to such good basic image processing tools. Ofc this is very slow, and also performs terrible on harder cases, but this is very interesting!!! Pre-AI image fill
